# 🏗️ SKILL: PRINCÍPIOS SOLID NO TYPESCRIPT

## 🎯 Objetivo
Garantir que o código seja robusto, fácil de testar e resiliente a mudanças através da aplicação dos 5 princípios SOLID.

## 📜 Os 5 Princípios
1. **S - Single Responsibility Principle:** Cada classe/função/componente deve ter um, e apenas um, motivo para mudar.
2. **O - Open/Closed Principle:** Entidades de software devem estar abertas para extensão, mas fechadas para modificação. Use polimorfismo e interfaces.
3. **L - Liskov Substitution Principle:** Objetos de uma superclasse devem poder ser substituídos por objetos de suas subclasses sem quebrar a aplicação.
4. **I - Interface Segregation Principle:** Clientes não devem ser forçados a depender de interfaces que não utilizam. Prefira interfaces pequenas e específicas.
5. **D - Dependency Inversion Principle:** Dependa de abstrações, não de implementações concretas. Use inversão de dependência para injetar serviços.

## 📂 Onde Aplicar
- Especialmente em `[package-core]` e `[package-renderer]`.
- Na definição de Use Cases e Repositórios.
