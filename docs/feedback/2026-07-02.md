## Feedback de Arquitectura

### Pontos Fortes

- **Separation of concerns** — Data collection, rendering, execução separados
- **Component composition** — Componentes reutilizáveis combinados em abas
- **Hook architecture** — useNavigate, useRefresh, useCommand independentes
- **Graceful degradation** — Fallback estático quando TUI não suporta raw mode
- **Accessibility** — Screen reader mode com aria-labels

### Áreas de Melhoria

1. **Estado global vs local** — `expandedItem` no useNavigate é global. Poderia ser por-aba
2. **Tipagem dos tabs** — Cada tab tem interface própria. Poderia ser genérica: `TabProps<T>`
3. **Error boundaries** — Não há ErrorBoundary no TUI. Crash num tab derruba todo o dashboard
4. **Testing library** — `ink-testing-library` é limitado. Componentes com `useOnWheel` não testáveis sem mock
5. **Performance** — `collectConsoleData()` chamada em cada refresh. Poderia usar cache

## Feedback de Código

### Padrões Positivos

- Nomes semânticos em inglês
- Componentes pequenos e focados (< 200 linhas)
- Props imutáveis com default values
- Early returns para estados vazios

### Correcções Sugeridas

1. **`use-command.ts`** — `generateId()` usa counter global. Usar `crypto.randomUUID()`
2. **`index.tsx`** — Handler de Enter usa `expandMap` hardcoded. Poderia ser dinâmico
3. **`commands.tsx`** — `globalIndex` incrementado durante render. Usar `useMemo`
